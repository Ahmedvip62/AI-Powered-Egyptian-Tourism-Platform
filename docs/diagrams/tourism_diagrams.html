Sequence Diagrams

4.1 — Visual RAG: Monument Image Recognition
sequenceDiagram actor User participant Frontend participant APIGateway as API Gateway participant AIService as AI Service participant CLIPEncoder as CLIP Encoder participant VectorDB as Vector DB (FAISS) participant LLM as Language Model User->>Frontend: Upload monument image Frontend->>Frontend: Preview & validate image Frontend->>APIGateway: POST /api/ai/analyze {image} APIGateway->>AIService: Forward request AIService->>CLIPEncoder: encode(image) CLIPEncoder-->>AIService: Return embedding [512-dim vector] AIService->>VectorDB: search(embedding, topK=5) VectorDB-->>AIService: Return top-5 similar monuments AIService->>LLM: generate(monuments_context, "Describe this monument") LLM-->>AIService: Return historical description AIService->>AIService: Store query + result in DB AIService-->>APIGateway: Return {description, confidence, monuments} APIGateway-->>Frontend: 200 OK {result} Frontend-->>User: Display historical description
4.2 — User Registration & Authentication
sequenceDiagram actor User participant Frontend participant API as Backend API (.NET) participant AuthService as Auth Service participant DB as PostgreSQL User->>Frontend: Fill registration form Frontend->>Frontend: Client-side validation Frontend->>API: POST /api/auth/register {name, email, password} API->>DB: Check if email exists DB-->>API: Not found API->>AuthService: hashPassword(password) AuthService-->>API: passwordHash API->>DB: INSERT user record DB-->>API: User created API->>AuthService: generateJWT(userId) AuthService-->>API: JWT token API-->>Frontend: 201 Created {token, user} Frontend->>Frontend: Store token in localStorage Frontend-->>User: Redirect to dashboard
